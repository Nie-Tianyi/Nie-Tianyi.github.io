一个小细节：为什么机器学习中总是习惯性的写成![image](images/逻辑回归’s Gradient/04fc4bfd1647f8ba0ca31a20fa7183fb.svg)，而不是![image](images/逻辑回归’s Gradient/717ec016851b38dc8e80a4bfe3d06442.svg)？因为在机器学习中，我们习惯性的将![image](images/逻辑回归’s Gradient/5e293cf754c9e3a9ad8af0b4c8b6971f.svg)和 ![image](images/逻辑回归’s Gradient/9bc21b9ec0ce99019d63120115080c9e.svg)视作形状为 ![image](images/逻辑回归’s Gradient/29cf1c9370447f71e2d6410da3551d32.svg)的列向量（column vector） ，于是![image](images/逻辑回归’s Gradient/f90656db63cc39782e3cb15df962c0b9.svg)实际上是一个形状![image](images/逻辑回归’s Gradient/29cf1c9370447f71e2d6410da3551d32.svg)，求转置后，成为一个形状为![image](images/逻辑回归’s Gradient/bd518381539e2259a8109c52beec3257.svg)的矩阵，然后乘以另一个形状为 ![image](images/逻辑回归’s Gradient/29cf1c9370447f71e2d6410da3551d32.svg)的矩阵。如果你足够了解矩阵乘法，那么你就会知道前者的计算结果是一个形状为![image](images/逻辑回归’s Gradient/5ce90cfd5dec8b917254288b23aebdb7.svg)的矩阵；而后者的计算结果是一个![image](images/逻辑回归’s Gradient/88a5675269d6d4cd8b9e592baf7a7adc.svg)的矩阵。在这里我们只想要得到一个标量，而不需要一个矩阵。

多项式逻辑回归

逻辑回归只能处理二分类问题，如果想要预测有多个标签的结果，单个逻辑回归模型就无能为力了。处理多标签分类问题时，最直观、最容易想到的办法是先将其中几个聚集为一类，分类，然后再在每个子类内继续分类。例如我们现在需要分类的结果有 [0, 1, 2, 3] 四类，那么我们可以先将0和1分为一类，3和4分为另一类，这样问题就简化成了多个二分类问题：

![](images/逻辑回归’s Gradient/1752349137934-5b2912db-74f7-47d0-9531-7f2f0acae18e.png)

在处理多分类的问题的时候，我们首先需要将预测标签编码成 one-hot 编码。例如在上例中，我们想要预测的结果有四类[0, 1, 2, 3]，但是我们实际的标签——假设这条数据的类别是2——则是[0, 0, 1, 0]，分别对应这[0, 1, 2, 3]。或者假设我们要预测的标签为 [红色， 绿色， 蓝色]，而这单条样本的标签为绿色，则我们需要将![image](images/逻辑回归’s Gradient/bf98c0ddcbe9c1e535f767c78c3aa813.svg)编码成 [0，1，0] （注意，我们所有的标签都需要遵循一样的次序）。

这里我们需要连续训练三个逻辑回归模型，如果需要分的类别越多，则需要训练的模型也越多。有更简单的方法吗？有的！在面对多分类问题时，我们可以使用多项式逻辑回归（Polynomial Logistic Regression）。多项式逻辑回归的核心是 Softmax 函数，我们用 Softmax 函数取代而二项式逻辑回归中的 Sigmoid 函数：

![image](images/逻辑回归’s Gradient/991302128cc8cc83c385ca18b5f07d6f.svg)

这里的![image](images/逻辑回归’s Gradient/da8ba3ff388a4a17525f3d51358aebd1.svg)表示每个类别的输入和其权重的乘积再加上偏置，即![image](images/逻辑回归’s Gradient/97c493e04d4b888043c22f94bba7b65a.svg)。假设我们有三个类别需要分类，分别是 [0, 1, 2] 三个类别：

  1. 我们首先初始化三组权重，分别是 ![image](images/逻辑回归’s Gradient/1f0b2c1a602f5a52f29bed25d3346115.svg)和![image](images/逻辑回归’s Gradient/ccd7921caa636e4b39239f38768dbc52.svg)、![image](images/逻辑回归’s Gradient/deb8257d4bfd5f75d99bc965a6088f98.svg)和![image](images/逻辑回归’s Gradient/1becf94a458da89497059c47a43c858a.svg)、![image](images/逻辑回归’s Gradient/7e5c604851fe7a8d19a97a5e63757c58.svg)和![image](images/逻辑回归’s Gradient/99b74c73e8f07cc892af007b651fce09.svg)；
  2. 接着我们计算权重和输入的乘积 ![image](images/逻辑回归’s Gradient/ba32bbf550e9b4770785882c14921e25.svg)、![image](images/逻辑回归’s Gradient/c6f04b14150a8bb1a100eb16b5d91c64.svg)、![image](images/逻辑回归’s Gradient/0e41bae649b03a29e8c613fa2f833063.svg);
  3. 最后我们通过 Softmax函数，计算出每个分类的概率![image](images/逻辑回归’s Gradient/39cdaec02c6d19112b5f870766d3e300.svg)、![image](images/逻辑回归’s Gradient/3da99999420353fda48faf744c988e15.svg)、![image](images/逻辑回归’s Gradient/9972f5a4aab8f6d74ec7220b90f1b686.svg)。这样我们就能得到三个概率，分别对应着预测为0、1、2三类的概率，并且![image](images/逻辑回归’s Gradient/fb1dd5af48470897a55b8cecb310d818.svg)。

我们使用softmax函数主要是为了将预测的结果归一化为三个加起来等于1的概率，那为什么使用以 ![image](images/逻辑回归’s Gradient/c9a277d40d3a0b3afb85cdb557908945.svg)为底的 Softmax 函数，而不是直接使用![image](images/逻辑回归’s Gradient/3a962584124a21a32d4757332498aaa0.svg)呢？后者不也可以将多个输出归一到和为1的多个概率吗？使用以 ![image](images/逻辑回归’s Gradient/c9a277d40d3a0b3afb85cdb557908945.svg) 为底的指数函数的目的主要有两个：一个是可以保证结果中不会出现负数，二个是可以利用指数爆炸的性质，放大多个预测结果之间的差异，使其更贴合我们期望的“其中一个类别的预测结果趋近于1，而其他几个类别的预测结果接近于0”的预测结果。至于为什么底是![image](images/逻辑回归’s Gradient/c9a277d40d3a0b3afb85cdb557908945.svg)?当然是为了我们后面计算梯度的时候求导方便啦！

这个时候我们得到了三个概率，那我们该怎么计算梯度，更新我们模型的权重参数呢？在多分类问题中，我们沿用逻辑回归中的损失函数 ![image](images/逻辑回归’s Gradient/30154f57030912fcbdcf346d646a82c6.svg)。假设数据的真实标签是0，那么我们期望我们的模型的输出为 [1, 0, 0]（分别对应![image](images/逻辑回归’s Gradient/c5940a3acff4f38f793a8ce19bdb8690.svg)![image](images/逻辑回归’s Gradient/091a44b6adf69464fa153ab4fb6b1d64.svg)![image](images/逻辑回归’s Gradient/764c1c88fba43c220e26e27403bf676f.svg)），但是我们模型实际输出为[0.34, 0.33, 0.33]，那么我们的模型这一轮训练的损失为![image](images/逻辑回归’s Gradient/65a313bfe1a63f7d6ac7f7d3d054640d.svg)。后面两项为0，这意味着在多项式分类中，我们只关心标签为真的项的损失。

计算多项式逻辑回归的梯度

跟之前求导梯度不一样的是，我们现在一共有![image](images/逻辑回归’s Gradient/38a3f4d664b7a723d138f9d57be0c783.svg)组权重，并且每组权重内有![image](images/逻辑回归’s Gradient/4760e2f007e23d820825ba241c47ce3b.svg)个权重以及一个偏置![image](images/逻辑回归’s Gradient/d29c2e5f4926e5b0e9a95305650f6e54.svg)。（假设我们预测的结果一共有![image](images/逻辑回归’s Gradient/38a3f4d664b7a723d138f9d57be0c783.svg)类，并且每条数据的维度为![image](images/逻辑回归’s Gradient/4760e2f007e23d820825ba241c47ce3b.svg)维）。

![image](images/逻辑回归’s Gradient/b787123144017b9367197b79be0de230.svg)

跟之前不同的是，之前的权重只有一个维度，而这里的权重成为了一个二维的矩阵，我们接下来要分别求出这个矩阵中每一个参数的梯度。在计算Softmax函数梯度之前，我们要根据标签![image](images/逻辑回归’s Gradient/bf98c0ddcbe9c1e535f767c78c3aa813.svg)的值分成两种情况，分别分成![image](images/逻辑回归’s Gradient/1951f9656e5e28af5edb9c326a2bfa91.svg) 的时候和 ![image](images/逻辑回归’s Gradient/e8cbef78c5f630a10a4ebd954dfb6649.svg)的时候。因为当![image](images/逻辑回归’s Gradient/0e30338d8bdf1ce3a92ee33809a7ae88.svg)的时候，Softmax函数的分子内包含我们要求的参数的偏导；而![image](images/逻辑回归’s Gradient/5ef5eebdaedab8a0d31a54532a9f9ef0.svg) 的时候，Softmax函数的分子内不包含我们要求的参数的梯度，因此我们可以将分子上的式子视作一个常数。

由于我们需要根据标签的值分成多种情况求导，当预测标签的值不一样的时候，我们计算各个参数的梯度的方式也不一样。所以和之前不一样的是，我们需要先计算单个样本上的梯度，再将所有梯度总和起来，最后将总和起来的梯度更新到权重上去——而不是将所有样本总和在一起批量求得梯度。
